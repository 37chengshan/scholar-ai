### 阶段 2: 上传5篇论文

**测试ID**: UAT-002  
**测试名称**: 批量论文上传  
**优先级**: P0

#### 推荐测试论文（不同领域）:

1. **计算机视觉**: Attention Is All You Need (Transformer)
2. **自然语言处理**: BERT: Pre-training of Deep Bidirectional Transformers
3. **机器学习**: ImageNet Classification with Deep Convolutional Neural Networks (AlexNet)
4. **强化学习**: Mastering the Game of Go (AlphaGo)
5. **多模态**: CLIP: Learning Transferable Visual Models From Natural Language Supervision

#### 测试步骤:

| 步骤 | 操作 | 预期结果 | 验证点 |
|-----|------|---------|--------|
| 2.1 | 访问 /upload 页面 | 上传页面加载 | 拖拽区域可见 |
| 2.2 | 拖拽或选择PDF文件 | 文件被选中 | 显示文件名和大小 |
| 2.3 | 点击"开始上传" | 上传进度显示 | 进度条动画 |
| 2.4 | 等待解析完成 | 显示解析结果 | 标题、作者、摘要提取 |
| 2.5 | 确认保存 | 论文保存到库 | 跳转到library或显示成功 |
| 2.6 | 重复2.1-2.5四次 | 5篇论文上传完成 | library显示5篇论文 |

#### API 端点:
```
# 获取上传URL
POST /api/papers/upload-url

# 上传文件（本地存储）
POST /api/upload/local/:storageKey

# 创建论文记录
POST /api/papers
```

## 🐛 常见问题排查

### 登录失败
```bash
# 检查用户是否存在
docker exec scholarai-postgres psql -U scholarai -d scholarai -c "SELECT email FROM users;"

# 重置密码
docker exec scholarai-postgres psql -U scholarai -d scholarai -c "UPDATE users SET password_hash='new_hash' WHERE email='test@example.com';"
```

### 上传失败
```bash
# 检查存储目录
ls -la scholar-ai/uploads/

# 检查Python服务日志
tail -f scholar-ai/backend-python/backend-python-test.log
```

### Chat无响应
```bash
# 检查Python AI服务
curl http://localhost:8000/

# 检查AI服务配置
echo $ZHIPU_API_KEY
```
