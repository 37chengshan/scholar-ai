# v2 与 v2.1（Parse Once Reuse）

## 定义
- v2：20 篇快速联调版，默认按 `prepare_raw_base -> build:raw -> build:rule -> build:llm -> eval:retrieval -> eval:answer` 执行。
- v2.1：解析复用版，若已存在 raw base，则默认跳过解析，仅重建 raw/rule/llm 三个可对照数据集。

## 为什么只解析一次
旧流程每个 stage 都会重新读 PDF、重新分页、重新切块，耗时和成本都高。

新流程把解析和切块前移为一次性产物：
1. `prepare_raw_base.py` 只负责解析与切块并持久化。
2. `build_stage_variant.py` 只消费 raw chunks 做轻加工和索引。

## 产物结构
- `artifacts/benchmarks/{profile}/raw_base/parsed_pages.jsonl`
- `artifacts/benchmarks/{profile}/raw_base/raw_chunks.jsonl`
- `artifacts/benchmarks/{profile}/raw_base/raw_chunks_manifest.json`
- `artifacts/benchmarks/{profile}/raw_base/paper_stage_timing.jsonl`

## 三个可对照数据集
- raw：`paper_contents_v2_qwen_v2_raw_v2` / `paper_contents_v2_qwen_v2_raw_v2_1`
- rule：`paper_contents_v2_qwen_v2_rule_v2` / `paper_contents_v2_qwen_v2_rule_v2_1`
- llm：`paper_contents_v2_qwen_v2_llm_v2` / `paper_contents_v2_qwen_v2_llm_v2_1`

每条 stage 数据都包含：
- `stage`
- `source_chunk_id`
- `raw_text`
- `contextual_prefix`
- `content_data`

## 运行命令
- v2：`./scripts/evals/run_academic_rag_v2.sh --dataset-profile v2`
- v2.1：`./scripts/evals/run_academic_rag_v2_1.sh`
