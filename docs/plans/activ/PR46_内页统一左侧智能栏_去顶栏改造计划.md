# PR46 内页统一左侧智能栏与去顶栏改造计划

> 创建日期: 2026-04-22  
> 范围: `apps/web` 登录后内部页面  
> 目标: 取消当前分散的顶部栏/页头体系，收敛为 GPT 风格的统一左侧智能工作台

---

## 一、改造目标

将当前登录后内部页面从“全局侧栏 + 页面页头 + 页面内二级栏 + 页面内说明条”的多层壳结构，改造成：

- 统一左侧导航栏
- 单一主画布
- 按需展开的右侧辅助面板

核心原则：

1. 去掉顶部大栏位，首屏直接进入任务本体。
2. 左侧栏只承担一级导航、对话和知识库文件夹。
3. 中央区域只保留当前任务最重要的内容。
4. 页面具体功能、上下文和辅助操作统一转移到右侧可收起面板。

---

## 二、当前核心问题

### 2.1 壳层重复

- 全局壳层在 `apps/web/src/app/components/Layout.tsx`
- Chat 自有顶部条在 `apps/web/src/features/chat/components/ChatHeader.tsx`
- Search 自有左来源栏与右分析栏在 `apps/web/src/features/search/components/SearchWorkspace.tsx`
- Read 自有顶部工具条、左目录和右侧 inspector 在 `apps/web/src/app/pages/Read.tsx`

结果是每个页面都像独立产品，无法形成统一的“AI 工作台”体验。

### 2.2 主内容出现过晚

- Layout 对非 Chat 页额外增加大页头。
- Chat 在消息流之前又叠加 session 标题栏、状态条、workflow 壳层。
- Search 在检索结果之外保留双侧 chrome。

结果是页面垂直空间浪费明显，信息密度虽然高，但重点不集中。

### 2.3 左侧栏职责混杂

当前左侧栏同时混入了：

- 页面说明
- 当前页面状态
- 上下文摘要
- 最近工作解读

这会导致左栏越来越像“第二主内容区”，削弱 GPT 风格工作台的简洁性。

---

## 三、目标信息架构

### 3.1 全局结构

统一为：

- 左侧导航栏
- 中央主画布
- 可选右侧抽屉

不再保留：

- 全局顶部导航栏
- 非 Chat 页大页头
- 页面级重复说明条
- 左侧栏里的页面说明/上下文卡片

### 3.2 左侧智能栏分区

从上到下固定为：

1. 品牌区
2. 主动作区
3. 工作区切换区
4. 对话列表区
5. 知识库文件夹区
6. 账号与设置区

左侧栏不再承担：

- 页面说明
- 页面级功能按钮组
- 当前模式摘要
- 分析/过滤/辅助操作面板

### 3.3 页面策略

#### Chat

- 去掉独立顶部标题条
- 左侧只保留会话列表
- 主区第一屏直接看到消息流和输入框
- Workflow 状态条压缩为轻量状态带
- 具体上下文与运行详情放进右侧可收起面板

#### Search

- 左侧只保留一级导航，不再保留 Search 专属左来源栏
- 主区保留搜索框与结果
- 来源、分析、筛选统一放入右侧分析区，默认关闭
- 结果卡操作改为常显

#### Read

- 左侧不再承担章节目录
- 右 inspector 默认关闭
- 顶部工具条压缩为悬浮式轻工具带
- PDF 主画布成为唯一视觉焦点
- 章节目录、笔记、总结集中到右侧可收起区

#### Knowledge Base

- 拆掉重型 sticky 顶栏
- 保留搜索 / 新建 / 视图切换三大一级动作
- 批量操作只在选中状态出现
- 统计信息与详细功能放入右侧抽屉
- 左侧只保留知识库文件夹入口

#### Notes

- 修复阻断性路由问题
- 形成稳定的“左树右编”工作流
- 取消大页头与重复说明文字
- 左侧文件树相关信息收敛到“知识库文件夹/笔记入口”，具体筛选和详情进入右侧抽屉

## 四、实施分期

### Phase 1: 统一壳层基础改造

目标：先把“去顶栏”方向落实到基础框架。

- `apps/web/src/app/components/Layout.tsx`
- `apps/web/src/features/chat/workspace/ChatWorkspaceV2.tsx`
