### 目标
API contract 第二轮收口：冻结 imports/chat/session 的对外契约，消除文档与实现漂移。

### 主要改动（保留原范围）
- 对齐 docs/architecture/api-contract.md
- 对齐 docs/domain/resources.md
- 对齐 apps/web/src/services/importApi.ts
- 对齐 packages/sdk/src/chat/stream.ts
- 对齐 apps/api/app/models/chat.py
- 对齐 apps/api/app/api/session.py

### 执行补充
- 明确 `chat stream` 请求体对 `mode/scope` 的支持状态、字段类型、默认值与兼容行为。
- 明确 `session messages` 的 `total/order/pagination` 语义：
  - `total` 指 session 全量消息总数，不是当前页数量。
  - `order` 明确排序方向（默认建议 `desc`）。
  - `pagination` 明确是否有更多数据及偏移量语义。
- 对齐 SDK 与前端 service 的类型定义，避免“后端 envelope 与前端数组”语义漂移。

无。

### 验收标准
- 文档、SDK、前端 service、后端 request/response 模型字段完全一致。
- `chat stream` 请求体是否支持 `mode/scope` 明确。
- `session messages` 的 `total/order/pagination` 语义明确。

### 最小验证
- `cd apps/web && npm run type-check`
- `cd apps/api && pytest -q tests/unit/test_services.py --maxfail=1`
